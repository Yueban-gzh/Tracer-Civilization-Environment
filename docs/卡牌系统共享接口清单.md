# 卡牌系统共享接口清单

供做卡牌系统的同学参考：项目中可与卡牌模块共同使用的基础函数、类型和接口。


## 一、可直接共享的类型与数据结构

**CardId** `include/Common/Types.hpp` - 卡牌唯一标识（如 "strike", "card_001"），直接 include

**InstanceId** `include/Common/Types.hpp` - 牌组内实例唯一 id（区分多张同 id 牌），直接 include

**StatusId** `include/Common/Types.hpp` - 状态效果 id（如 "weak", "vulnerable"），直接 include

**CardInstance** `include/CardSystem/CardSystem.hpp` - 卡牌实例：instanceId, id, temporary，直接 include

**CardData** `include/DataLayer/DataLayer.hpp` - 卡牌静态数据：name, cost, cardType, description 等，直接 include

**CardType** `include/DataLayer/DataLayer.hpp` - Attack / Skill / Power / Status / Curse，直接 include

**CardColor** `include/DataLayer/DataLayer.hpp` - Red / Blue / Green / Purple / Colorless / Curse，直接 include

**Rarity** `include/DataLayer/DataLayer.hpp` - Common / Uncommon / Rare / Special，直接 include

**StatusInstance** `include/BattleCoreRefactor/Types.hpp` - 状态实例：id, stacks, duration，直接 include


## 二、CardSystem 核心函数（可直接复用）

init_master_deck(card_ids) - 初始化永久牌组

get_master_deck() - 获取永久牌组

get_master_deck_card_ids() - 获取永久牌组 CardId 列表

add_to_master_deck(id) - 永久加入一张牌

remove_from_master_deck(instance_id) - 从永久牌组移除

upgrade_card_in_master_deck(instance_id) - 升级永久牌组中的牌

init_deck(initial_card_ids) - 战斗开始：初始化四牌堆

draw_cards(n) - 抽 n 张牌

get_hand() - 获取手牌

remove_from_hand(hand_index) - 从手牌移除并返回

add_to_hand(card) - 加入手牌

add_to_discard(card) - 加入弃牌堆

add_to_exhaust(card) - 加入消耗堆

shuffle_discard_into_draw() - 弃牌堆洗回抽牌堆

add_to_deck(card) - 加入抽牌堆

remove_from_deck(instance_id) - 从牌堆移除指定实例

get_deck_size() / get_discard_size() / get_exhaust_size() - 各牌堆大小

get_draw_pile() / get_discard_pile() / get_exhaust_pile() - 各牌堆只读引用

upgrade_card_in_deck(instance_id) - 升级战斗中的牌

generate_to_hand(id) - 生成临时牌到手牌

generate_to_draw_pile(id) - 生成临时牌到抽牌堆

generate_to_discard_pile(id) - 生成临时牌到弃牌堆

generate_to_exhaust_pile(id) - 生成临时牌到消耗堆

register_card_effect(id, fn) - 注册 CardId 到效果函数

execute_effect(id, ctx) - 执行卡牌效果

has_effect_registered(id) - 检查是否已注册效果

依赖：CardSystem 构造需注入 GetCardByIdFn（从数据层按 id 获取 CardData）


## 三、EffectContext 接口（卡牌效果执行必需）

卡牌效果函数签名为 void(EffectContext& ctx)，执行时需通过 EffectContext 调用战斗逻辑。

deal_damage_to_monster(idx, amount) - 对怪物造成伤害（考虑格挡、易伤等），攻击牌

deal_damage_to_all_monsters(base) - 对全体怪物造成伤害，范围攻击

deal_damage_to_player(amount) - 对玩家造成伤害，自伤牌

deal_damage_to_player_ignoring_block(amount) - 对玩家无视格挡伤害，御血术等

deal_damage_to_monster_ignoring_block(idx, amount) - 对怪物无视格挡伤害，穿透伤害

add_block_to_player(amount) - 给玩家加格挡，防御牌

add_block_to_monster(idx, amount) - 给怪物加格挡，给敌人加盾

get_effective_damage_dealt_by_player(base, idx) - 玩家对怪物的有效伤害（含力量、易伤等），计算实际伤害

get_effective_block_for_player(base) - 玩家有效格挡（含敏捷等），计算实际格挡

get_player_block() - 获取玩家当前格挡，全身撞击等

apply_status_to_monster(idx, id, stacks, duration) - 对怪物施加状态，虚弱、易伤

apply_status_to_player(id, stacks, duration) - 对玩家施加状态，力量、敏捷

apply_status_to_all_monsters(id, stacks, duration) - 对全体怪物施加状态，闪电霹雳等

get_status_stacks_on_monster(idx, id) - 怪物某状态层数，飞踢判断易伤

get_status_stacks_on_player(id) - 玩家某状态层数，重刃计算力量加成

generate_to_discard_pile(id) - 生成卡牌到弃牌堆，愤怒等

draw_cards(n) - 抽牌，剑柄打击等

add_energy_to_player(amount) - 加能量，飞踢等

ctx.target_monster_index - 当前目标怪物下标（-1 表示无目标），效果内读取目标

说明：EffectContext 由 BattleEngine 填充，内部持有 BattleEngine*。若同学独立实现卡牌系统，需要提供类似的上下文接口，或对接本项目的 BattleEngine。


## 四、数据层接口（卡牌静态数据）

get_card_by_id(CardId) - 在 include/DataLayer/DataLayer.hpp，按 id 获取 CardData*

CardData 结构 - id, name, cardType, cost, color, rarity, description, exhaust, ethereal, innate, retain, unplayable, requiresTarget

cards.json - 在 data/cards.json，卡牌数据 JSON 格式，可共用或扩展


## 五、状态效果相关（StatusModifiers）

get_status_stacks(list, id) - BattleEngine 静态方法，从状态列表中取某 id 的层数

reduce_status_stacks(list, id, amount) - BattleEngine 静态方法，减少状态层数（回合末 tick）

create_player_status_modifiers(player) - 在 StatusModifiers.hpp，根据玩家状态生成 modifier

create_monster_status_modifiers(monsters) - 在 StatusModifiers.hpp，根据怪物状态生成 modifier

状态效果（weak、vulnerable、strength 等）的数值计算在 StatusModifiers.cpp 中实现，卡牌效果通过 EffectContext 间接使用。
