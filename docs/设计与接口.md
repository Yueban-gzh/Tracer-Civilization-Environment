# 设计与接口（阶段 0 定稿后即按此实现）

> 分工与排期见 [课设分工与排期.md](课设分工与排期.md)。本文档集中：MVP、数据表、数据结构映射、公共类型、各模块接口、依赖与 Mock。

## 一、MVP 范围与一局流程

| 项目 | 取值 |
|------|------|
| 地图 | 单塔 5～6 层，每层 3～4 节点，DAG，起点到终点可达 |
| 节点类型 | 战斗、事件、休息、商店 |
| 卡牌 | 20～30 张（攻击/防御/技能/辅助） |
| 敌人 | 6～8 普通怪 + 1 Boss |
| 商店/事件树/休息 | 都做（商店：买牌+删一张；休息：回血 or 升级一张牌） |

**一局流程**：初始化 → 循环「展示当前层节点 → 选节点 → 战斗/事件/休息/商店」→ 层推进 → 最后一层 Boss 胜则通关结算。

## 二、数据表格式（E 牵头定稿）

**卡牌表**：id, name, cardType(攻击/防御/技能/辅助), cost, rarity?, description?, effectType?, effectValue?

**怪物表**：id, name, isBoss, maxHp, intentPattern?, attackDamage?, blockAmount?

**事件表**：id, title, description, options（数组：每项 text + next 或 result）。事件树用一条记录内嵌 options 的树形结构即可。

数据文件可放 `data/`（如 cards.json、monsters.json、events.json）。

| 模块 | 数据结构 | 落点 |

## 五、各模块接口（仅列名与依赖）

**A - MapEngine**  
- 对外：`init_map`, `get_nodes_at_layer` → [{ nodeId, nodeType, contentId? }], `get_next_layer`, `is_boss_node`, `validate_reachable`（BFS/DFS）。  
- 依赖：无（节点类型可写死或从常量读）。

**B - BattleEngine**  
- 对外：`start_battle(monsterIds, playerState)`, `get_battle_state`, `play_card`, `end_turn`, `is_battle_over`, `get_reward_cards`。  
- 依赖 **C**：init_deck, draw_cards, get_hand, remove_from_hand, add_to_discard, shuffle_discard_into_draw, get_deck_size, get_discard_size。  
- 依赖 **E**：get_monster_by_id；可选 get_card_by_id。

**C - CardSystem**  
- 对外：init_deck, draw_cards, get_hand, remove_from_hand, add_to_discard, shuffle_discard_into_draw, add_to_deck, remove_from_deck, get_deck_size, get_discard_size, upgrade_card_in_deck。  
- 依赖 **E**：get_card_by_id。

**D - EventEngine**  
- 对外：start_event, choose_option, get_event_result；open_shop, buy_card；rest_heal, rest_upgrade_card（或由主流程调 C）。  
- 依赖 **E**：get_event_by_id。依赖 **C**：add_to_deck, remove_from_deck, upgrade_card_in_deck。

**E - DataLayer**  
- 对外：get_card_by_id, get_monster_by_id, get_event_by_id；sort_cards_by_rarity, sort_leaderboard；load_cards/load_monsters/load_events。  
- 依赖：无。

## 六、依赖与 Mock（并行开发用）

| 开发方 | 依赖谁 | Mock 要点 |
|--------|--------|-----------|
| B | C、E | Mock C：init_deck/draw_cards/get_hand/remove_from_hand/add_to_discard/shuffle/get_*_size 用内存列表实现，返回固定手牌或空实现。Mock E：get_monster_by_id 返回固定血量/意图。 |
| C | E | Mock E：get_card_by_id 始终返回同一张牌（含 cost、name）。 |
| D | E、C | Mock E：get_event_by_id 返回一棵写死的事件树。Mock C：add_to_deck/remove_from_deck/upgrade 空实现或简单列表操作。 |
| A | 无（或 E） | 可不依赖 E，用常量写死节点类型与 contentId。 |

联调时用真实现替换 Mock 即可。
