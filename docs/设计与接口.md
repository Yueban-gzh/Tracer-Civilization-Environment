> 分工与排期见 [课设分工与排期.md](课设分工与排期.md)。本文档集中：MVP、数据表、数据结构映射、公共类型、各模块接口、依赖与 Mock。

## 一、MVP 范围与一局流程

| 项目 | 取值 |
|------|------|
| 地图 | 单塔 5～6 层，每层 3～4 节点，DAG，起点到终点可达 |
| 节点类型 | 战斗、事件、休息、商店 |
| 卡牌 | 20～30 张（攻击/防御/技能/辅助） |
| 敌人 | 6～8 普通怪 + 1 Boss |
| 商店/事件树/休息 | 都做（商店：买牌+删一张；休息：回血 or 升级一张牌） |

**一局流程**：初始化 → 循环「展示当前层节点 → 选节点 → 战斗/事件/休息/商店」→ 层推进 → 最后一层 Boss 胜则通关结算。

## 二、数据表格式（E 牵头定稿）

**卡牌表**：id, name, cardType, cost, rarity, description；**词条** exhaust、ethereal、innate、retain、unplayable。**是否升级**由 id 区分（如 strike / strike+ 两条数据）。**牌实例**含牌组内唯一 InstanceId + CardId，多张同 id 牌靠 InstanceId 区分。**效果不由 E 提供**：C 通过「CardId → 效果函数」查找并调用（见 [BC模块设计与接口](BC模块设计与接口.md)）。E 牵头定稿卡牌表字段。

**怪物表**：id, name, isBoss（或 type 普通/精英/Boss）, maxHp。**意图、攻击伤害、格挡量不由 E 提供**：B 通过「怪物 id → 行为函数」查找并调用，每只怪对应一个函数（见 [BC模块设计与接口](BC模块设计与接口.md)）。

**事件表**：id, title, description, options（数组：每项 text + next 或 result）。**事件树定死为一条记录内嵌 options**：有 `next` 表示跳转到对应 EventId 的子节点，有 `result` 表示该选项的结局（如 { "type": "gold", "value": 20 }），由 D 解析。

数据文件放 `data/`（cards.json、monsters.json、events.json），编码 UTF-8，路径相对可执行文件或项目根在阶段 1 前约定。

### 类型与示例（E 阶段 0 定稿、阶段 1 填表用）

**卡牌表**  
- cardType：枚举 `"attack"|"defend"|"skill"|"support"`。  
- rarity：枚举 `"common"|"uncommon"|"rare"`，排序时顺序为 common < uncommon < rare。  
- effectType / effectValue：MVP 可简化为 description 字符串 + 少量数值（如 damage, block）；复杂效果后续用 effectType 枚举 + effectValue 扩展。  
- 示例一行：`{ "id": "card_001", "name": "与子同袍", "cardType": "attack", "cost": 1, "rarity": "common", "description": "造成8点伤害", "effectType": "damage", "effectValue": 8 }`  

**怪物表**  
- intentPattern：字符串或字符串数组，表示每回合意图顺序（如 `"attack"`、`"block"`、`"buff"`），B 按顺序或随机取。  
- attackDamage / blockAmount：单值或与 intent 对应；MVP 可用单值。  
- 示例一行：`{ "id": "monster_001", "name": "残鼎之灵", "isBoss": false, "maxHp": 30, "intentPattern": "attack,block", "attackDamage": 8, "blockAmount": 5 }`  

**事件表**  
- options 每项：`{ "text": "选项描述", "next": "event_002" }` 或 `{ "text": "选项描述", "result": { "type": "gold", "value": 15 } }`。  
- 示例：一个事件两条 options，一条 next 到子事件，一条 result 结束。

| 模块 | 数据结构 | 落点 |
|------|----------|------|

## 四、公共类型

| 类型 | 含义 |
|------|------|
| NodeType | Battle / Event / Rest / Shop |
| CardId, MonsterId, EventId | 卡牌/怪物/事件唯一 id，**统一用 string**（如 "card_001", "monster_boss_1"） |
| NodeId | 地图节点 id（如层号+节点下标，与层号约定一致） |
| 层号 | **从 0 开始**，0 表示第一层；A 的 get_nodes_at_layer(0) 为第一层节点 |

## 五、各模块接口（仅列名与依赖）

**A - MapEngine**  
- 对外：`init_map`, `get_nodes_at_layer` → [{ nodeId, nodeType, contentId? }], `get_next_layer`, `is_boss_node`, `validate_reachable`（BFS/DFS）。  
- 依赖：无（节点类型可写死或从常量读）。

**B - BattleEngine**  
- 对外：`start_battle(monsterIds, playerState)`, `get_battle_state`, `play_card`, `end_turn`, `is_battle_over`, `get_reward_cards`。  
- 依赖 **C**：init_deck, draw_cards, get_hand, remove_from_hand, add_to_discard, shuffle_discard_into_draw, get_deck_size, get_discard_size。  
- 依赖 **E**：get_monster_by_id；可选 get_card_by_id。

**C - CardSystem**  
- 对外：init_deck, draw_cards, get_hand, remove_from_hand, add_to_discard, shuffle_discard_into_draw, add_to_deck, remove_from_deck, get_deck_size, get_discard_size, upgrade_card_in_deck。  
- 依赖 **E**：get_card_by_id。

**D - EventEngine**（详见 [D模块设计与接口.md](D模块设计与接口.md)）
- 对外：start_event, choose_option, get_current_event, get_event_result；open_shop, buy_card；rest_heal, rest_upgrade_card；先序/层序遍历（traverse_preorder, traverse_level_order）。
- 依赖 **E**：get_event_by_id。依赖 **C**：add_to_deck, remove_from_deck, upgrade_card_in_deck。

**E - DataLayer**  
- 对外：get_card_by_id, get_monster_by_id, get_event_by_id；sort_cards_by_rarity, sort_leaderboard；load_cards/load_monsters/load_events。  
- 依赖：无。  
- **约定**：load_* 由主流程或 GameMain 启动时各调一次（或 E 首次 get_*_by_id 时懒加载）；id 不存在时返回 null/空 optional，不抛异常；sort_cards_by_rarity 入参为 CardId 数组或卡牌引用数组，出参为按 rarity（common → uncommon → rare）排序后的同类型数组；sort_leaderboard 可选，结构约定为 { playerId, score, time? } 列表按 score 降序。

## 六、依赖与 Mock（并行开发用）

| 开发方 | 依赖谁 | Mock 要点 |
|--------|--------|-----------|
| B | C、E | Mock C：init_deck/draw_cards/get_hand/remove_from_hand/add_to_discard/shuffle/get_*_size 用内存列表实现，返回固定手牌或空实现。Mock E：get_monster_by_id 返回固定血量/意图。 |
| C | E | Mock E：get_card_by_id 始终返回同一张牌（含 cost、name）。 |
| D | E、C | Mock E：get_event_by_id 返回一棵写死的事件树。Mock C：add_to_deck/remove_from_deck/upgrade 空实现或简单列表操作。 |
| A | 无（或 E） | 可不依赖 E，用常量写死节点类型与 contentId。 |

联调时用真实现替换 Mock 即可。
